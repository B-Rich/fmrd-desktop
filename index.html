FMRD-Desktop 1.0.2

The Football Match Result Database (FMRD) stores and maintains the match result data needed to support football (soccer) research activities for the benefit of the football analytics community. The current version of the database models league competitions.

The Football Match Result Database has the following features:

  • Maintains data on matches that occur within football league competitions, whether among national teams or domestic clubs.
  • Maintains complete top-level data on the football match, including
    • Match date
    • Competition name
    • Competition stage
    • Participating teams
    • Host venues
    • Environmental conditions such as kickoff time, temperature, and weather conditions at kickoff, halftime, and fulltime
  • Maintains complete information on participating personnel, such as
    • Players
    • Managers
    • Match referees
  • Maintains complete information on macro-events that occur during a match
    • Goals
    • Penalties
    • Disciplinary incidents
    • Substitutions
    • Forced or tactical position changes

Dependencies

  • Python 2.5-2.7 (3.0 not supported)
  • Nokia Qt 4 toolkit
  • PyQt libraries
  • PostgreSQL 8.x
  • Perl (DBI package)

Development

The code repository is located at http://github.com/hhamilton-src/fmrd-desktop. You can clone the project in Git with the following command:

$ git clone git://github.com/hhamilton-src/fmrd-desktop

The code and supporting documentation can be downloaded by all, but in order to contribute to code development you'll have to create a Github account.

Installation

  • Check out the most recent release of the application from the GitHub repository into a local working directory.
  • Set up a local Postgres server on your machine, and obtain a username and password. Assistance from your system administrator may be required.
  • Go to the sql/ directory and run the following script:

    $ installdb-fmrd.sh db_name

  • After the database is formed you will be asked to enter username and password. The script will then load the validation tables of the database will be loaded.
  • Run the data-entry application from the command-line:

    $ ./fmrd-main.pyw
