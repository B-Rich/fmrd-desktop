FMRD-Desktop 1.3.0/1.4.0

The Football Match Result Database (FMRD) is a database schema that facilitates the storage and maintenance of match result data needed to support football (soccer) research activities for the benefit of the football analytics community. Version 1.3.0 models league, knockout, and hybrid competitions for club teams and Version 1.4.0 does the same for national teams.

The Football Match Result Database has the following features:

  • Maintains data on matches that occur within football league competitions, whether among national teams or clubs.
  • Maintains complete top-level data on the football match, including
    • Match date
    • Competition name
    • Competition phase
    • Phase-specific identifiers
      • League phase: Matchday
      • Group phase: Round, Group, Matchday
      • Knockout phase: Round, Matchday
    • Competition matchday
    • Participating teams
    • Host venues
    • Environmental conditions such as kickoff time, temperature, and weather conditions at kickoff, halftime, and fulltime
  • Maintains complete information on participating personnel, such as
    • Players
    • Managers
    • Match referees
  • Maintains complete information on macro-events that occur during a match
    • Goals
    • Penalties
    • Disciplinary incidents
    • Substitutions
    • Forced or tactical position changes
    • Penalty kick shootouts

Build Dependencies

  • Python 2.5-2.7 (3.0 not supported)
  • Nokia Qt 4 toolkit
  • PyQt libraries

Database Dependencies

  • PostgreSQL 8.x+ OR SQLite3 3.6.19+
  • Perl (DBI package)

The code and supporting documentation can be downloaded by all, but in order to contribute to code development you will have to create a Github account.

Setup

  • Check out the most recent release of the application from the Github code repository into a local working directory.
  • Also (or alternatively), download the most recent release of the application executable from the FTP site and decompress into a local working directory.
  • The FMRD-Desktop application can be run with either a PostgreSQL or SQLite backend.
  • For use with PostgreSQL:
    • Set up a local Postgres server on your machine, and obtain a username and password. Assistance from your system administrator may be required.
    • Go to the sql/ directory and run the following script:

      $ installdb-fmrd.sh db_name

    • After the database has been formed you will be asked to enter username and password, followed by the maximum number of league or shootout rounds and the type of group identifiers (alpha or numeric). The script will then load the validation tables of the database.
  • For use with SQLite3:
    • Go to the sql/ directory and run the following script:

      $ installdb-fmrd-sqlite.sh db_name.db

      The database file must have a '*.db' extension.

      After the database has been formed you will be asked to enter the maximum number of league or shootout rounds and the type of group identifiers (alpha or numeric). The script will then load the validation tables of the database.

  • You can run the data-entry application in one of two ways:
    • From the command-line in the code/pyqt/ directory:

      $ ./fmrd-main.pyw

    • Or, double-click on the appropriate FMRD-Desktop executable.
